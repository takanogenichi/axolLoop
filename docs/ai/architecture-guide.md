
## 全体アーキテクチャ

これは日本の求人プラットフォーム向けデータ変換・登録システムとして機能するLaravel 12アプリケーションです。マルチパッケージアーキテクチャを採用しています：

### コアパッケージ（`packages/` ディレクトリ）
- **LaratapCores** - データベース操作、スキーマ変更、共通抽象化を含むコアユーティリティとユースケース
- **AxolCommon** - アプリケーション全体で共有されるエンティティ、Enum、サービス（クライアント解析、データベース命名、IPサービス）
- **AxolHome** - admin、client、convert操作向けの広範なデータベースマイグレーションとシーダーを持つHomeモジュール
- **AxolPackage** - adminおよびclientデータベーススキーマを含むビジネスロジックの大部分を持つメインパッケージ

### メインアプリケーション構造（`app/` ディレクトリ）
- **Services/AxolRegisters** - 複数の求人プラットフォームに対応するデータ登録システム：
    - `Mynavi/` - マイナビ求人プラットフォーム連携
    - `Airwork/` - Airworkプラットフォーム連携
    - `CareerTasu/` - キャリタス連携
- **Services/Downloader** - Webスクレイピング機能を持つプラットフォーム固有のデータダウンローダー
- **Services/ConvertJobs** - 非同期データ処理用のキュージョブ
- **Models** - アクセスレベル別に整理（Admin、Client、HomeAdmin、HomeClient、HomeConvert）

### 主要機能領域
1. **データ変換パイプライン** - 各種求人プラットフォームのデータを標準フォーマットに変換
2. **マルチプラットフォーム連携** - マイナビ、Airwork、キャリタスプラットフォームを専用のダウンローダーとコンバーターで処理
3. **住所・学校名正規化** - 日本の郵便番号、都道府県、大学名の標準化
4. **重複検出** - 異なるデータソース間の高度な重複チェック
5. **キュー処理** - ダウンロードとデータ変換のバックグラウンドジョブ処理

## テスト構成
- `tests/Unit/` にユニットテスト（登録機能やユーティリティのテストを含む）
- `tests/Feature/` にAPIおよび結合テスト
- `tests/Unit/Register/Navi/Datas/` に各プラットフォーム向けの豊富なテストデータ
- 外部APIテスト用のモックサービス

## 設定に関する注意事項
- Docker Compose（MySQL 8、PHP 8.4、Laravel 12）を使用
- PHPStanはベースライン付きで設定済み（段階的な型安全性の改善のため）
- 環境別（admin、client、convert）のカスタムデータベースシーディングシステム
- League Flysystemによるファイルストレージ用S3連携

## 開発ワークフロー
1. 初期セットアップには `make init` を使用
2. 開発コンテナに入るには `make php` を使用
3. コミット前に `make test` と `make stan` を実行
4. データベース変更時はマイグレーションとシーダーを適用するために `make init_db` の実行が必要
